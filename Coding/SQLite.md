### Первая строка: Запись данных

`db.prepare('REPLACE INTO users (id, mode) VALUES (?, ?)').run(id, mode);`

1. **`db`**: Это объект твоей базы данных (твой "шкаф" с информацией).

2. **`.prepare(...)`**: Команда «Подготовься». Ты как бы заранее диктуешь базе шаблон команды, которую она должна выполнить.

3. **`REPLACE INTO`**: Это "умная" вставка. Она говорит: «Положи данные в таблицу. Если пользователь с таким `id` уже есть — **обнови** его режим (mode). Если его нет — **создай** новую строчку».

4. **`users`**: Имя таблицы, в которой мы работаем.

5. **`(id, mode)`**: Список колонок в таблице, куда мы хотим записать данные.

6. **`VALUES (?, ?)`**: Значения. Знаки вопроса — это **заглушки** (безопасные "дырки"). Мы не пишем данные прямо в текст команды, чтобы хакеры не могли взломать базу через текстовое поле.

7. **`.run(id, mode)`**: Команда «Пли!». Только сейчас база берет реальные значения из твоих переменных `id` и `mode`, подставляет их вместо знаков вопроса и сохраняет в файл.

### Вторая строка: Чтение данных

`db.prepare('SELECT mode FROM users WHERE id = ?').get(id);`

1. **`SELECT mode`**: Команда «Выбери (достань) мне значение из колонки `mode`». Нам не нужен весь пользователь, только его режим.

2. **`FROM users`**: Указываем, из какой таблицы забирать (из таблицы "users").

3. **`WHERE id = ?`**: Это фильтр. «Достань режим **только для того** пользователя, чей `id` равен вот этому числу». Без этого условия база вернула бы режимы вообще всех людей.

4. **`.get(id)`**: Для команд `SELECT` используется `.get()`. Это значит: «Найди **одну** первую попавшуюся строчку, которая подходит под условие, и верни её мне». В скобках мы передаем реальный ID пользователя вместо знака вопроса.
