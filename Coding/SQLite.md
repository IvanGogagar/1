### Создание базы

```js
db.prepare(`
  CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY, 
    mode TEXT DEFAULT 'textMode', 
    authorized BOOLEAN DEFAULT false
  )
`).run();
```
### Первая строка: Запись данных

`db.prepare('REPLACE INTO users (id, mode) VALUES (?, ?)').run(id, mode);`

1. **`db`**: Это объект твоей базы данных (твой "шкаф" с информацией).

2. **`.prepare(...)`**: Команда «Подготовься». Ты как бы заранее диктуешь базе шаблон команды, которую она должна выполнить.

3. **`REPLACE INTO`**: Это "умная" вставка. Она говорит: «Положи данные в таблицу. Если пользователь с таким `id` уже есть — **обнови** его режим (mode). Если его нет — **создай** новую строчку».

4. **`users`**: Имя таблицы, в которой мы работаем.

5. **`(id, mode)`**: Список колонок в таблице, куда мы хотим записать данные.

6. **`VALUES (?, ?)`**: Значения. Знаки вопроса — это **заглушки** (безопасные "дырки"). Мы не пишем данные прямо в текст команды, чтобы хакеры не могли взломать базу через текстовое поле.

7. **`.run(id, mode)`**: Команда «Пли!». Только сейчас база берет реальные значения из твоих переменных `id` и `mode`, подставляет их вместо знаков вопроса и сохраняет в файл.


---

### Вторая строка: Чтение данных

`db.prepare('SELECT mode FROM users WHERE id = ?').get(id);`

1. **`SELECT mode`**: Команда «Выбери (достань) мне значение из колонки `mode`». Нам не нужен весь пользователь, только его режим.

2. **`FROM users`**: Указываем, из какой таблицы забирать (из таблицы "users").

3. **`WHERE id = ?`**: Это фильтр. «Достань режим **только для того** пользователя, чей `id` равен вот этому числу». Без этого условия база вернула бы режимы вообще всех людей.

4. **`.get(id)`**: Для команд `SELECT` используется `.get()`. Это значит: «Найди **одну** первую попавшуюся строчку, которая подходит под условие, и верни её мне». В скобках мы передаем реальный ID пользователя вместо знака вопроса.

### Третья строка

`const totalCountObj = db.prepare('SELECT COUNT(*) as count FROM users').get();`

- `COUNT(*)` — Считает, сколько всего людей в таблице. Это нужно, чтобы бот понял, есть ли смысл рисовать кнопку «Вперед».

- `.get()` — Возвращает одну строку (объект), а не массив.
### 1. Чтение данных (Вопросы к базе)

Это те команды, которые ты используешь с методом `.get()` (одна строка) или `.all()` (весь список).

- **`SELECT`** — «Выбери». Главное слово. После него пишешь колонки, которые нужны (или `*`, если нужны все).
    
- **`FROM`** — «Из». Указывает, из какой таблицы брать данные.
    
- **`WHERE`** — «Где». Фильтр. Без него команда применится ко всем сразу.
    
- **`ORDER BY`** — «Сортируй по». Например, `ORDER BY id DESC` покажет последних юзеров в начале.
    
- **`LIMIT`** — «Ограничь». Например, `LIMIT 5` достанет только первые 5 строк.

- `OFFSET ?` — Пропусти столько-то строк от начала.

> **Пример:** `SELECT mode FROM users WHERE id = ?` (Выбери режим из таблицы юзеров, где ID совпадает).

---

### 2. Запись и изменение (Действия)

Эти команды используются с методом `.run()`.

- **`INSERT INTO`** — «Вставь в». Добавляет новую строку.
    
- **`UPDATE`** — «Обнови». Меняет данные в уже существующих строках.
    
- **`SET`** — «Установи». Используется вместе с UPDATE, чтобы указать новое значение.
    
- **`DELETE FROM`** — «Удали из». Стирает строку. (Осторожно! Если забудешь `WHERE`, удалишь вообще всех).
    
- **`REPLACE INTO`** — «Замени в». Если такой ID уже есть — обновит, если нет — создаст.
    

---

### 3. Структура (Создание таблиц)

Обычно запускаются один раз при старте бота.

- **`CREATE TABLE`** — «Создай таблицу».
    
- **`IF NOT EXISTS`** — «Если её еще нет». Очень важная добавка, чтобы бот не выдавал ошибку при втором запуске.
    
- **`PRIMARY KEY`** — «Главный ключ». Пометка для колонки (обычно `id`), что она уникальна.
    
- **`DEFAULT`** — «По умолчанию». Значение, которое подставится само (например, `mode = 'textMode'`).
